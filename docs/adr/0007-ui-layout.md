# ADR 0007: UI 架构

## 背景

fitword 前端需要选择组件库和整体布局架构。

## 决策

### 使用 shadcn/ui sidebar 组件

用 shadcn sidebar（`SidebarProvider`、`SidebarMenuButton` 等）替代手写 `<aside>`。

**理由：**

- 自带折叠/展开、移动端响应式 sheet、键盘快捷键
- 与已有的 shadcn 组件（button、tabs 等）一致的数据属性 pattern（`data-slot`、`data-active`）
- 主题通过 CSS 变量统一控制，与主界面色调一致

### session 列表作为侧边栏主要导航

用 session 列表取代原来的「对话」「统计」两个 tab。

**理由：**

- 每个对话是独立的持久化单元，用户需要在多个对话间切换
- session 列表天然适合侧边栏垂直布局
- 「统计」退化为独立导航项，不与 session 平级

### 语言切换移入设置弹窗

侧边栏底部仅保留齿轮图标，点击打开设置 Dialog。

**理由：**

- 语言切换是低频操作，不应占侧边栏常驻空间
- 设置弹窗为后续配置项（如快捷键、主题）预留入口

### 移除欢迎消息

不再在聊天区顶部固定欢迎消息。

**理由：**

- 新用户第一个会话本身就是空白状态，不需要冗余提示
- session 恢复后欢迎消息没有意义
- 空状态文案在无 session 选中时居中显示即可

## 备选方案

| 方案                  | 优点     | 缺点                                        |
| --------------------- | -------- | ------------------------------------------- |
| 手写 `<aside>` 侧边栏 | 简单     | 无折叠、移动端适配需自己写                  |
| 两 tab（对话 + 统计） | 结构简单 | session 数量多时 tab 不够，无法管理多个对话 |
| 欢迎消息保留          | 新手引导 | 多 session 下冗余，恢复时无意义             |

## 后果

- 需要 session API（列表、归档、消息历史）
- 前端需要 session 状态管理（当前 session、session 列表、localStorage 持久化）
- 统计面板通过「统计」导航项进入，与对话视图互相独立
- 语言切换通过侧边栏底部齿轮图标 → 设置弹窗完成
- 详细 UI 规格见 `docs/prd/ui-layout.md`、`docs/prd/ui-settings.md`
