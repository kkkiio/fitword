# ADR 0007: UI Layout

## 状态

已采纳

## 背景

fitword 的 Web UI 需要承载三种交互模式（练习、写作、统计），通过侧边栏 tab 切换。需要定义布局结构和各 tab 下的交互行为。

## 决策

### 整体布局

```
┌──────────────────────────────────────────────┐
│  fitword（词感）                             │
├───────────┬──────────────────────────────────┤
│           │                                  │
│  📝 练习  │                                  │
│  ✍️  写作  │        主聊天区                  │
│  📊 统计  │                                  │
│           │                                  │
│           │                                  │
│           │                                  │
│           ├──────────────────────────────────┤
│           │  输入框                           │
└───────────┴──────────────────────────────────┘
```

- 左侧边栏：固定宽度，tab 切换
- 右侧主体：聊天区 + 底部输入框
- 输入框仅在聊天区下方，不跨到侧边栏

### Tab：练习

默认 tab。Agent 以出题为核心行为。

- 用户可自由聊天，Agent 在对话中主动出题（选择或填空）
- 聊天区中题目以卡片形式渲染：`____` 占位符替换为选项按钮或输入框
- 用户可以自然语言要求切换题型（"来几道填空题"）

### Tab：写作

写作评分模式。Agent 以评价为核心行为。

- 用户贴入文字，Agent 调用 `evaluate_writing` 展示评分卡片
- 评分卡片在聊天区中以独立卡片渲染：五维分数、主要问题、可替换词列表、改写版本
- 评分后 Agent 可顺势出题，出题卡片复用练习 tab 的渲染组件
- Agent 不做评分以外的主动出题

### Tab：统计

练习数据可视化。从 SQLite 读取。

- 展示内容：总题数、正确率趋势、薄弱类型分布、选择题 vs 填空题对比
- 以简洁图表呈现（柱状图、趋势线），不做复杂仪表盘
- 只读展示，无交互操作
- v0.1 不做日期筛选和导出

### Tab 切换行为

- 切换 tab 不丢失对话——Agent session 保持
- 切换到练习或写作时，Agent 感知当前模式，调整行为
- 统计 tab 独立于对话，切换回来时对话状态不变

## 备选方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 三 tab 侧边栏 | 模式清晰，用户知道当前在做什么 | 增加 UI 复杂度 |
| 单聊天区，Agent 全自适应 | UI 极简 | 用户不知道 Agent 当前行为倾向，容易混乱 |
| 无统计 tab，数据融入对话 | 少一个 tab | 数据可视化比文字描述直观 |

## 后果

- 引入侧边栏 + tab 的 UI 复杂度，但每个 tab 下的交互逻辑独立清晰
- 统计 tab 需要前端图表库（如 Recharts）
- 写作和练习的题目卡片可复用同一渲染组件
