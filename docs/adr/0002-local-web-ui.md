# ADR 0002: 本地 Web UI

## 背景

fitword 需要用户交互界面来承载 chatbot 对话和题目展示。原始 PRD 提到"CLI 或本地 Web UI"两种可能性，需要做出选择。

## 决策

使用 **本地 Web UI**，参考 roundtable 的形态：轻量、本地启动、浏览器访问。

- 前端：React + Tailwind CSS
- 后端：本地 HTTP 服务（Hono 或类似轻量框架）
- 启动方式：命令行启动后访问终端输出的 localhost 地址
- 无远程部署，仅 localhost 访问

## 备选方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| CLI / TUI | 极简，终端即用 | 无法渲染选择题按钮、填空题输入框等富交互组件；选择/填空的 UI 体验差 |
| 本地 Web UI | 富交互自由，聊天体验好 | 需要浏览器，多一层依赖 |
| Electron 桌面应用 | 独立窗口体验 | v0.1 过度工程化 |

## 后果

- 选择题的 4 按钮、填空题的内联输入框、评分卡片的视觉呈现均可自由设计
- 本地 HTTP 服务 + pi SDK Agent 运行在同一进程
- 后续如需迁移到 Electron 或其他容器，Web UI 代码可直接复用
- 需处理浏览器 tab 关闭后 Agent 会话的挂起与恢复
