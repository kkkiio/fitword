# Product Requirements: Practice Model

## 概述

题目练习是 fitword 的基础练习机制，包含两种题目格式：**选择题**（辨认）和**填空题**（产出）。写作评分是最高难度的综合练习，单独在评分模型中定义。交互以 chatbot 对话为主线——Agent 与用户自然聊天，在合适的时机呈现题目，用户作答后 Agent 给予反馈并继续对话。

---

## 题目类型

### 选择题

语境 + 4 个候选词，用户从中选择。核心能力是**辨认**——在给定选项中识别最合适的词。门槛低，适合热身和新话题入门。

### 填空题

语境 + 一个留空，用户自己打出答案。核心能力是**产出**——不依赖选项提示，从自己的词汇库中主动调用。更接近真实表达场景，难度更高。v0.1 仅支持单个留空。

Agent 在出题时自主选择用选择题还是填空题，可根据用户水平、题型特点、对话节奏来定。一般来说：新话题用选择题热身，熟悉话题或用户主动要求时用填空题加难度。

---

## 知识类别

以下五种类别适用于选择题和填空题：

| 类别 | 考查内容 | 选择题示例语境 | 填空题示例语境 |
|---|---|---|---|
| 名词 | 东西、结构、现象叫什么 | 靠窗的长条桌子叫什么？[4选1] | 靠窗的长条桌子叫____。 |
| 动词 | 动作如何描述更精准 | 他在会议上____了观点。[4选1] | 他在会议上____了观点。 |
| 形容词 | 性质、质感、风格如何表达 | 这部电影的节奏很____。[4选1] | 这部电影的节奏很____。 |
| 逻辑词 | 因果、条件、让步、递进如何连接 | 他很早到了会场，____错过了致辞。[4选1] | 他很早到了会场，____错过了致辞。 |
| 领域词 | 特定领域的专业表达 | 咖啡的____很突出。[4选1] | 咖啡的____很突出。 |

## 交互模型

### 对话驱动

1. 用户以任意方式开启对话。
2. Agent 以自然语言回应。
3. 用户可随时以自然语言切换话题。
4. Agent 在合适的时机向用户呈现题目（选择题或填空题）。
5. 用户作答。
6. Agent 给出反馈，然后继续对话。

### 示例交互（选择题）

```
用户: 最近我在写周报时总觉得用词太干了，想练练
Agent: 写周报确实容易落入套路。你在周报里通常会描述哪类内容？
用户: 主要是进度汇报
Agent: 那咱们来一道动词选择题，看看描述进度时用哪个词更精准。

[呈现选择题]
  语境: 项目第一阶段的开发已经____，下周进入测试。
  A. 完成  B. 告一段落  C. 收尾  D. 结束

用户: B
Agent: 对，B 最好。比"完成"多了"阶段性停顿、后续还有"的含义。
      "结束"太绝对，"收尾"暗示还有零星工作。
```

### 示例交互（填空题）

```
Agent: 刚才那道选择题你答得不错。试试同一个语境，但这次不给选项——

[呈现填空题]
  上周的需求变更打乱了排期，项目不得不往后____。

用户: 推迟
Agent: "推迟"可以，但语境里是"打乱了排期"的被动调整，
      用"顺延"会更准确——暗示不是主动推迟，而是被推到后面。
```

## 出题

Agent 自主生成题目内容并在对话中呈现。出题时机由 Agent 根据对话上下文判断，用户也可主动要求（"来几道填空题"）。

Agent 可自主选择出选择题还是填空题，通常：新话题用选择题热身，用户水平提升或主动要求时用填空题加难度。

---

## 反馈规则

### 选择题反馈

1. **判定**：正确 / 错误
2. **正确答案**：明确标注
3. **选项辨析**：逐项解释每个选项在语境中的适配度

### 填空题反馈

1. **评价**：不简单说"对/错"——用户的答案可能不是最佳但能接受
2. **对比**：将用户答案与更贴切的表达对比，说明差异和各自的语境适配场景
3. **建议**：如果用户答案不理想，引导用户思考更贴切的词，而非直接否定

填空题的核心原则：**用户写出一个能用的词就该肯定，同时教一个更好的。**

答题质量用于后续反馈和统计，分为三档：不可用、可用但不佳、好。选择题只产生错误或正确两种结果；填空题可落在三档中的任意一档。

## 数据记录

完成作答的题目数据会被本地保存。记录内容包括：

- 题目格式（选择题 / 填空题）
- 知识类别、语境、话题标签
- 选择题：候选词、正确答案、用户选择、答题质量
- 填空题：用户输入、Agent 对答案质量的判断
- 时间戳

## Agent Memory 与统计

Agent 能感知用户的练习历史。当前为"基础统计感知 + 只读轻量统计页"级别。

### 感知内容

- 各知识类别的可用率、优质率和待打磨率
- 选择题 vs 填空题的答题质量对比
- 话题偏好和练习频次
- 总体题数、总体可用率、优质率和待打磨率

### 统计页

- 展示总体题数、可用率、优质率和待打磨率
- 待打磨率表示未达到"好"的答题比例
- 展示按知识类别的质量分布，突出薄弱类型
- 展示选择题 vs 填空题的质量对比
- 可轻量展示写作评分概览；详细评分反馈仍留在对应对话中
- 只读展示，不做日期筛选、导出或手动编辑

### 行为规范

- Agent 可在对话中自然提及统计信息
- 可据此调整出题策略（如弱项多练、填空比例逐步提高）
- 不在对话中主动倾倒完整统计面板；统计摘要由用户进入统计页查看

---

## 边界条件

1. 每道题仅属于一种格式 + 一种知识类别。
2. 填空题仅包含一个留空。
3. 用户答完当前题并看到反馈后，Agent 才能出下一道题。
4. 用户可随时以自然语言终止练习。
5. Agent 不应在用户未回应的情况下连续出题。
6. 退出后数据不丢失，恢复对话时可继续。
