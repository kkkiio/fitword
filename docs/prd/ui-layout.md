# UI 布局

fitword 的 Web UI 承载对话交互和练习数据可视化。练习和写作不是两种模式——评分后 Agent 可顺势出题，两者在同一对话流中交替。唯一区分是输入框的发送意图不同。

## 整体布局

有 session 时（选中了某个对话）：

```
┌─────────────────┬──────────────────────────────┐

│  词感·表达练习   │  │                      │    │
│                 │  │  Agent: 这里用"阶段性  │    │
│  对话  +       │  │  收尾"会更好...       │    │
│ ┌─────────────┐ │  │                      │    │
│ │ ▶ 项目汇报..│ │  └──────────────────────┘    │
│ └─────────────┘ │  ┌──────────────────────┐    │
│ ┌─────────────┐ │  │  阶段性收尾           │    │
│ │   咖啡风味..│ │  └──────────────────────┘    │
│ └─────────────┘ │                              │
│ ┌─────────────┐ │                              │
│ │   技术方案..│ │                              │
│ └─────────────┘ │                              │
                              │
│                 ├──────────────────────────────┤
│  统计           │  ┌────────────────────┐       │
│                 │  │ 输入你的消息...     │       │
│                 │  └────────────────────┘       │
│  ⚙             │  ○ 写作评分           [发送]  │

```

- 侧边栏右侧有一道 1px 分隔线
- 当前选中的 session 高亮（▶ 或加粗）
- 未选中的 session 灰色文字，hover 才显示归档按钮
- 「对话」标题右侧有 + 图标按钮，点击新建对话
- 「统计」是独立导航项，样式类似 session 项但用不同图标
- 侧边栏底部齿轮图标 → 打开设置弹窗（语言切换在其中）
- 右侧主区域：顶部聊天历史 + 底部固定输入框

### 对话 Session 管理

session 列表仅在侧边栏「对话」区域，不进入主聊天区。

**产品规则：草稿对话与正式 session**

- 点击「对话」标题右侧的 + 图标后，产品进入一个新的空白草稿对话。
- 草稿对话没有标题，不出现在侧边栏，不参与刷新后的恢复，也不占用历史记录。
- 用户在草稿对话里发送第一条消息后，才生成一个正式 session。
- 正式 session 生成后立即出现在侧边栏，并自动成为当前选中项。
- 如果用户进入草稿对话后没有发送任何消息，再切换到其他 session 或刷新页面，不保留这个草稿。
- 侧边栏只展示已经有首条用户消息的正式 session。

**侧边栏结构：**

```
┌─────────────────┐
│  fitword        │  ← 品牌区
│  词感·表达练习   │    圆角高亮
├─────────────────┤
│  对话  +       │  ← 标题行，右侧 + 新建
│                 │     + 只切换右侧到欢迎界面，不产生会话记录
│ ┌─────────────┐ │
│ │ ▶ 项目汇报..│ │  ← 当前选中
│ │         归档│ │     hover 时右侧出现归档
│ └─────────────┘ │
│ ┌─────────────┐ │
│ │   咖啡风味..│ │  ← 未选中，hover 出现背景色
│ └─────────────┘ │
│ ┌─────────────┐ │
│ │   技术方案..│ │     session 在首条消息发送后才出现
│ └─────────────┘ │
│                 │
│ （无更多）       │  ← 没有"新对话"占位项
├─────────────────┤
│  📊 统计        │  ← 独立 nav item，不等同于 session
├─────────────────┤
│  ⚙             │  ← 齿轮图标，打开设置弹窗
└─────────────────┘
```

**交互规则：**

- 每次对话是一个 session，刷新页面后仍可恢复
- 标题：取该 session 第一条用户消息的前 20 个字符，不足则全取
- session 按更新时间倒序排列
- 选中 session → 右侧主区域：还原历史消息 + 底部固定输入框
- 新建对话 → 点击「对话」旁的 + 图标，右侧切换到欢迎界面，侧边栏不增加新项；发送首条消息后才生成 session
- 归档 session → 确认弹窗，确认后从侧边栏移除；如果归档的是当前 session，右侧切到空白草稿状态
- 刷新页面后优先恢复上次选中的 session；如果该 session 已不存在，则显示空状态

### 对话视图

右侧主体区域，根据是否有选中 session 展示不同内容。

**空状态（无 session 选中，含欢迎界面）：**

点击「+ 新建对话」或首次进入时显示：

```
┌──────────────────────────────────────────────┐
│                                              │
│                                              │
│              fitword                         │
│              词感 · 表达练习                  │
│                                              │
│  ┌────────────────────────────┐              │
│  │ 说一个想练的话题...          │              │
│  └────────────────────────────┘              │
│  ○ 写作评分                      ┌──────┐   │
│                                │ 发送 │   │
│                                └──────┘   │
└──────────────────────────────────────────────┘
```

- 品牌名居中显示
- 底部固定输入框（textarea + 写作评分开关 + 发送按钮，开关与发送按钮同行）
- 发送首条消息后：session 出现在侧边栏，当前界面变为活跃对话

**活跃对话（有 session 选中）：**

```
┌──────────────────────────────────────────────┐
│                                              │
│  ┌──────────────────────────┐                │
│  │  聊聊项目结项汇报的表达?    │ ← 用户消息    │
│  └──────────────────────────┘                │
│  ┌──────────────────────────┐                │
│  │  好的。项目尾声常用"收尾"   │ ← Agent 消息  │
│  │  、"验收"、"复盘"...       │               │
│  └──────────────────────────┘                │
│  ┌──────────────────────────┐                │
│  │  【选择题】靠窗那排适合    │ ← 题目卡片    │
│  │   临时办公的长条桌...      │               │
│  │   A. 吧台  B. 工位        │               │
│  │   C. 展台  D. 柜台        │               │
│  └──────────────────────────┘                │
│                                              │
├──────────────────────────────────────────────┤
│  ┌────────────────────────────┐              │
│  │ 输入你的消息...              │              │
│  └────────────────────────────┘              │
│  ○ 写作评分                      ┌──────┐   │
│                                │ 发送 │   │
│                                └──────┘   │
└──────────────────────────────────────────────┘
```

- Agent 行为：默认以练习为主，在对话中主动出题（选择或填空）
- 用户可自然聊天、切换话题、指定题目格式
- 题目以卡片形式渲染在聊天区中（与普通消息视觉区分）
- 每次发送消息关联到当前 session，历史消息从服务端加载

**输入框**：

- 底部固定
- 输入框下方有一个「写作评分」开关（Switch），与发送按钮同行，关闭时发送普通消息，打开时进入评分模式
- 开关打开后：textarea 变高
- 发送按钮始终显示「发送」，根据开关状态决定 `intent`（普通消息 vs score）
- 评分后开关自动恢复关闭状态

```
默认状态:           ┌─────────────────────┐
                    │ 输入你的消息...      │
                    └─────────────────────┘
                    ○ 写作评分       [发送]

评分状态:           ┌─────────────────────┐
                    │ 输入你的消息...      │
                    └─────────────────────┘
                    ● 写作评分       [发送]
```

### 题目卡片渲染

选择或填空题在聊天区中以卡片渲染，`____` 替换为交互组件：

- 选择题：4 个选项按钮，用户点击即提交
- 填空题：文本输入框 + 确认按钮
- 卡片背景与普通聊天消息区分，视觉上可识别为"题目"

### 评分卡片渲染

评分结果在聊天区中以卡片渲染：

- 总分 + 五维柱状
- 主要问题文字
- 可替换词列表（原词 → 替换词 + 理由）
- 改写版本全文
