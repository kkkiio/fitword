# Product Requirements: Scoring Model

## 概述

写作评分是 fitword 的综合表达练习，也是难度最高的一种练习。用户输入一段自己写的文字，Agent 从五个维度评分并给出具体改进建议。与选择题练习一样，评分以 chatbot 对话为交互主线；Agent 仅在用户明确请求写作评分时进行评分。

---

## 交互模型

### 对话驱动

1. 用户明确请求写作评分，并提交一段待评价文字。
2. Agent 分析文字并生成五维评分、问题说明、替换建议和改写版本。
3. 界面将评分结果以自然对话口吻呈现，而非冷冰冰的报告。

### 示例交互

```
用户: 帮我看看这段周报：
      "本周完成了大部分开发任务，整体进度正常，下周继续推进。"

Agent: 好。几点观察：
      准确度还行，但具体度偏弱——“大部分任务”“正常推进”太笼统了。
      试试改成"完成了 3 个接口联调和 2 个 bug 修复，下周进入测试阶段"会清晰很多。

      评分如下（1-5）：
      准确度 4 | 具体度 2 | 自然度 4 | 结构 3 | 语域 4

      想听每个维度的详细解释吗？
```

---

## 评分结果

Agent 负责对用户文字进行评价，并给出结构化评分结果。

### 输入内容

- 用户提供的原始文本

### 输出内容

- **总分**：1-5
- **五维评分**：每个维度 1-5 分
- **主要问题**：用词粗、概念模糊、逻辑松、语气不自然等
- **可替换词建议**：2-3 个，每个包含原词、建议替换词、理由
- **改写版本**：一段基于原文的改写，保持原意但提升表达质量

### 频率限制

单次对话中，评分频率由用户请求决定，不做硬性限制。Agent 不主动邀请写作评分，也不评价普通消息中的文字。

---

## 评分维度

| 维度 | 含义 | 低分特征（1-2） | 高分特征（4-5） |
|---|---|---|---|
| 准确度 | 词是否贴合语境 | 词不达意，用词偏离原意 | 每个词恰到好处，难以替换 |
| 具体度 | 是否过于笼统 | 大量"很好""不错""很多事情" | 有具体细节、数据、实例支撑 |
| 自然度 | 是否像真实口语或自然书面语 | 生硬、翻译腔、套路化 | 读起来像真人说的话或写的文 |
| 结构 | 表达逻辑是否清楚 | 句子间跳跃，因果关系模糊 | 逻辑清晰，层次分明 |
| 语域 | 是否符合场景语体 | 正式场合用口语，或聊天中太书面 | 与场景匹配的语体风格 |

---

## 反馈格式

Agent 返回评分结果时遵循：

1. **先说重点**：直接指出最突出的问题维度
2. **给出例子**：引用原文中的具体词句说明问题
3. **提供改进**：可替换词建议和改写版本
4. **评分做参考**：分数是辅助，不建议用户盯着数字看

---

## 数据记录

写作评分是一种练习，评分记录需要进入本地练习历史。记录内容包括原文、五维评分、主要问题、可替换词建议、改写版本和时间戳。

---

## 边界条件

1. Agent 仅在用户明确请求写作评分并提供待评价文字后进行评分。
2. 文字过短（如一两句）时，Agent 可建议用户提供更充分的素材。
3. 评分结果可讨论——用户可追问某个维度或不同意 Agent 的判断。
4. 评分后 Agent 可顺势出题（如发现用户某个知识类别偏弱），并根据语境判断生成选择题或填空题。用户完成作答后进入练习历史。
