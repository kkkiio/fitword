# Product Requirements: Scoring Model

## 概述

写作评分是 fitword 的第二个核心练习机制。用户输入一段自己写的文字，Agent 从五个维度评分并给出具体改进建议。与选择题练习一样，评分以 chatbot 对话为交互主线——Agent 可主动或在用户请求时调用评分工具。

---

## 交互模型

### 对话驱动

1. 用户在对话中贴入一段文字，或 Agent 在聊天中询问用户是否有想被评价的文字。
2. Agent 调用评分工具。
3. Agent 将评分结果以自然对话口吻呈现，而非冷冰冰的报告。

### 示例交互

```
用户: 帮我看看这段周报：
      "本周完成了大部分开发任务，整体进度正常，下周继续推进。"

Agent: 好。几点观察：
      准确度还行，但具体度偏弱——“大部分任务”“正常推进”太笼统了。
      试试改成"完成了 3 个接口联调和 2 个 bug 修复，下周进入测试阶段"会清晰很多。

      评分如下（1-5）：
      准确度 4 | 具体度 2 | 自然度 4 | 结构 3 | 语域 4

      想听每个维度的详细解释吗？
```

---

## 评分工具

Agent 通过评分工具对用户文字进行评价。

### 输入

- 用户提供的原始文本

### 输出

- **总分**：1-5
- **五维评分**：每个维度 1-5 分
- **主要问题**：用词粗、概念模糊、逻辑松、语气不自然等
- **可替换词建议**：2-3 个，每个包含原词、建议替换词、理由
- **改写版本**：一段基于原文的改写，保持原意但提升表达质量

### 频率限制

单次对话中，评分工具调用频率由 Agent 自然控制，不做硬性限制。Agent 不应在用户未请求或未贴文字时主动评分。

---

## 评分维度

| 维度 | 含义 | 低分特征（1-2） | 高分特征（4-5） |
|---|---|---|---|
| 准确度 | 词是否贴合语境 | 词不达意，用词偏离原意 | 每个词恰到好处，难以替换 |
| 具体度 | 是否过于笼统 | 大量"很好""不错""很多事情" | 有具体细节、数据、实例支撑 |
| 自然度 | 是否像真实口语或自然书面语 | 生硬、翻译腔、套路化 | 读起来像真人说的话或写的文 |
| 结构 | 表达逻辑是否清楚 | 句子间跳跃，因果关系模糊 | 逻辑清晰，层次分明 |
| 语域 | 是否符合场景语体 | 正式场合用口语，或聊天中太书面 | 与场景匹配的语体风格 |

---

## 反馈格式

Agent 返回评分结果时遵循：

1. **先说重点**：直接指出最突出的问题维度
2. **给出例子**：引用原文中的具体词句说明问题
3. **提供改进**：可替换词建议和改写版本
4. **评分做参考**：分数是辅助，不建议用户盯着数字看

---

## 数据记录

评分记录后续可存入 SQLite，v0.1 暂不定义评分存储 schema，先验证交互模式。

---

## 边界条件

1. 评分工具仅在被调用时工作，不自动扫描用户文字。
2. 文字过短（如一两句）时，Agent 可建议用户提供更充分的素材。
3. 评分结果可讨论——用户可追问某个维度或不同意 Agent 的判断。
4. 评分后 Agent 可顺势出题（如发现用户某个知识类别偏弱），出题即存，无需预存未做题目。
